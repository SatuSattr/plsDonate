# Comprehensive Setup Guide

This guide provides a deep dive into setting up **plsDonate-Express**, from initial installation to your first successful donation.

## 1. The "Why" Behind plsDonate
Most server owners struggle with official payment gateways because they require extensive legal documentation (country-specific documents, business registration, etc.). **plsDonate** solves this by leveraging **Tako.id**—a popular Indonesian streamer donation platform with a built-in API.

Instead of building and maintaining your own donation website, **plsDonate-Express** handles everything in-game. It's faster for the owner and more seamless for the player.

### How the Donation Process Works
1.  **Request:** A player types `/donate <amount> <email> <method> [message]`.
2.  **Review:** A confirmation dialog appears in-game for the player to review their request.
3.  **Confirm:** Once the player clicks **[Yes]**, the plugin sends an email receipt with a "Pay Now" link.
4.  **Pay:** The player opens their email (on phone or PC), clicks the link, and pays via Tako.id.
5.  **Fulfill:** The plugin detects the payment, broadcasts it to the server, and executes rewards automatically.

## 2. Supported Versions & Installation
The **Express** version is currently the only available version of plsDonate. It is a streamlined, high-performance version that manages its own webhook server.
- **Tested Versions:** Minecraft 1.21 through 1.21.1 (the entire 1.21 branch is supported).
- **Installation:** Download the `.jar` file and place it in your `plugins/` folder.

## 3. First-Time Startup
When you first start the server, you will see several **red warning messages** in the console. 
- *Examples:* `Tako.id Webhook Token is not set!`, `All SMTP hosts are using default credentials!`

**Don't worry!** This is normal. It's simply the plugin informing you that you need to fill out your credentials in the configuration files.

### Understanding the Files
The plugin generates the following in `plugins/plsDonate-Express/`:
- **`config.yml`**: General settings and API credentials.
- **`database.db`**: The transaction ledger. **Never edit this file manually.**
- **`triggers.yml`**: Where you define reward commands for donations.
- **`lang/en-US.yml`**: All player-facing text and messages.
- **`templates/payment.html`**: The HTML body for email receipts.

## 4. Tako.id Integration
Open `config.yml` and find the `tako` section. You will need credentials from your Tako.id Dashboard:
1.  **Creator Name:** Copy your handle from your profile (e.g., `sattr`).
2.  **API Key:** Go to *Integrations > API Integration* and copy the **Private API Key**.
3.  **Webhook Token:** Go to *Integrations > Webhook Notifications* and copy the **Webhook Token**.

## 5. Network & Port Configuration
The plugin needs an open port to "hear" when a donation is completed. 
- **Default Port:** `21172`.

### Pterodactyl & Hosting Panels
If you use a panel like Pterodactyl, you **cannot** use any port you want. 
1.  Go to your panel's **Network** settings.
2.  Assign an **Additional Port** to your server.
3.  Copy this port and update the `port:` setting in your `config.yml`.

### Local Environments (Windows)
If running on your PC, you can use the default port, but you will need a tool like **ngrok** to expose your local port to the public internet so Tako.id can reach you.

## 6. Email Setup (Gmail SMTP)
To send payment links, we recommend a Gmail account.
1.  Enable **2-Step Verification** on your Google Account.
2.  Search for **"App passwords"** in your account settings.
3.  Create a password named `plsDonate-Express` and copy the 16-character code.
4.  In `config.yml` under `email.hosts.1`, set:
    - `user`: Your Gmail address.
    - `pass`: The 16-character App Password code.

## 7. Reload & Test
Once everything is filled out, run `/pdn reload` in your console.

### Testing the Webhook
In your Tako.id dashboard, set your Webhook URL to:
`http://<YOUR_SERVER_IP>:<YOUR_PORT>/plsdonate`

If configured correctly, you can use `/pdn simulate fake <player> <amount>` to test your triggers without spending money!

## 8. Real-World Reward Example
You can set up your `triggers.yml` to give currency or ranks. 
- **Example:** If a player donates **Rp40.000**, you can use the `CoinsEngine` plugin command to give them **40 Coins**. They can then spend those coins in your server's shop!
