# Commands & Permissions

Manage your donation system using these built-in commands.

## Player Commands

### `/donate`
The primary command for players to initiate a donation.
- **Usage:** `/donate <amount> <email> <method> [message]`
- **Methods:** `qris`, `gopay` (min 10k), `paypal` (min 50k)
- **Example:** `/donate 10000 hello@gmail.com qris Support for the server!`

## Admin Commands
All admin commands require the `plsdonate.admin` permission.

### `/plsdonate reload` (alias: `/pdn reload`)
Reloads all configuration files, triggers, and the webhook listener.

### `/plsdonate transaction`
Manage the donation ledger.
- `list [page]`: View all recorded transactions.
- `info <id>`: View detailed information (TX-ID, Checksum, etc.).
- `delete <id>`: Remove a transaction record.
- `clear <player|all>`: Mass delete records.

### `/plsdonate simulate`
Used for testing your setup without spending real money.
- `fake <player> <amount>`: Triggers a "fake" donation that executes rewards immediately (Sandbox mode).
- `push <player> <amount>`: Pushes a "real" donation request to the ledger.

## Permissions Summary

| Permission | Description | Default |
| :--- | :--- | :--- |
| `plsdonate.admin` | Access to all administrative commands. | OP |
| `plsdonate.cooldown.bypass` | Bypass the donation command cooldown. | False |
